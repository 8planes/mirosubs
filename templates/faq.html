  1. {% trans 'What is Universal Subtitles?' %}

    {% trans "Our goal with Universal Subtitles is to make it easy for anyone to add subtitles, captions, or translate nearly any video on the web. Furthermore, we're building in collaborative tools, and hope to attract a large body of contributors to assist in the subtitling effort. The software we're building is free and open source and can be used and improved upon by anyone." %}

  2. {% trans 'How can I use Universal Subtitles?' %}

    {% blocktrans %}You're welcome to try the subtitling technology in our alpha demonstration. This demo doesn't yet allow you to save subtitles, but that feature will be included in the alpha community launch (targeted for July 2010).{% endblocktrans %}

  3. {% trans 'What are the benefits of subtitles?' %}

    {% trans 'Video subtitles are a gold mine of untapped potential.' %}

    • {% trans 'Captions make video more accessible for people with hearing disabilities' %}
    • {% trans "Translated subtitles make it possible for all of us to watch video in languages that we don't speak" %}
    • {% trans 'As an educator, storyteller, scientist, or anyone with a message, the potential size of your audience increases exponentially when you have subtitles and translations' %}

  4. {% trans 'Who is the Participatory Culture Foundation and why are they making Universal Subtitles?' %}

    {% trans 'The Participatory Culture Foundation is a 501c3 non-profit organization whose mission, through software and advocacy, is to build a fairer, more open, and more democratic media space. PCF saw a gap in the marketplace for more open and collaborative subtitling -- Universal Subtitles is an attempt to fill that need.' %}

  5. {% trans 'Will Universal Subtitles support closed captions for the deaf and hard of hearing?' %}

    {% trans 'Yes, we are working on a solution for making closed captions [including information about sounds and music], as well as subtitles/translations.' %}

  6. {% trans 'Will your subtitles support my language or characterset?' %}

    {% trans 'We will be working to support as many languages/charactersets as possible.' %}

  7. {% trans 'When will the tools, including this site, be available in my language?' %}

    {% trans "We're working towards having the subtitling widget and community website fully translatable, so the interfaces can be available in the user's native language." %}

  14. {% trans 'Where can I download subtitles in .SRT or other formats?' %}

    {% blocktrans %} When subtitles exist, you can download them in .SRT, .SSA, and TTML formats on the page for that language (example). Click the links in the lower right under the green "Edit Subtitles" button. {% endblocktrans %}

  15. {% blocktrans %}It says "Someone else is currently editing these subtitles" and I can't do anything. How long will this last for? Why does it happen?{% endblocktrans %}

    {% blocktrans %} The system currently only allows one user to edit subtitles for any given video at a time. So, if someone else is already editing subtitles for the video when you try to edit the subtitles, you'll get this message. {% endblocktrans %}

  16. {% trans "Why can't I change the timing on a translation?" %}

    {% blocktrans %} Right now, the timing information of a translation is linked to the timing of the original transcript. Soon we will let users "fork" the timing of translations, creating new versions that can have different timing or--potentially--a totally different number of subtitles. {% endblocktrans %}

  17. {% trans "How do old revisions work? What if somebody makes a change to my work that I don't approve of?" %}

    {% blocktrans %} Each set of subtitles or translation has a full revision history. Just click on the tab on the subtitle page in you will see a list of revisions. You can use the checkboxes on the left and the "compare revisions" button to see the difference between two revisions. Or you can click on any one revision, and edit based on that revision or roll back to that revision.

    If the current version is blank, but there are older revisions, it make sense to check those older revisions before starting from scratch, because there could be useful work in an older revision. {% endblocktrans %}
